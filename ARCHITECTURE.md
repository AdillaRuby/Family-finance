## 2. NAMING CONVENTIONS

### Files & Folders
- **Pages:** `page.tsx`
- **Layouts:** `layout.tsx`
- **Route groups:** `(auth)`, `(dashboard)` — parentheses mean "don't include in URL"
- **Components:** PascalCase file names matching export — `Button.tsx` exports `Button`
- **Utilities:** camelCase — `utils.ts`, `prisma.ts`
- **Feature folders:** kebab-case — `savings/`, `forgot-password/`

### Code
- **Components:** PascalCase — `Button`, `DashboardCard`, `TransactionList`
- **Hooks:** `use` prefix — `useTransactions`, `useAuth`, `useTheme`
- **Server Actions:** `async function` with descriptive names — `createTransaction`, `deleteUser`
- **Services:** camelCase — `transactionService.ts` exports `transactionService`
- **Types:** PascalCase for types/interfaces — `Transaction`, `User`, `CreateTransactionInput`
- **Zustand stores:** `use` prefix — `useAuthStore`, `useTransactionStore`

### Constants
- **UPPER_SNAKE_CASE** for true constants — `MAX_FILE_SIZE`, `DEFAULT_PAGE_SIZE`

## 3. NEXT.JS 16 BREAKING CHANGES (Applied)

**Reference:** Next.js 16.2.10 is installed (not v15 as spec originally stated).

### Key Changes from v15 → v16:
1. **Async Request APIs** — `params` and `searchParams` are now **Promises** and must be awaited:
   ```ts
   // Next.js 16
   export default async function Page(props: { params: Promise<{ id: string }> }) {
     const { id } = await props.params
   }
   ```

2. **`middleware` renamed to `proxy`** — `middleware.ts` → `proxy.ts` (Milestone 3)

3. **`revalidateTag` requires second argument** — `revalidateTag('posts', 'max')`

4. **Turbopack is default** — No need for `--turbopack` flag

5. **React 19.2** — Uses latest React Canary (19.2.4 installed)

6. **Prisma 7** — Connection URLs now passed via `adapter` in `PrismaClient` constructor, not in `schema.prisma`.

```ts
export const prisma = new PrismaClient({ adapter })
```

## 5. SERVER ACTION VS ROUTE HANDLER DECISIONS

**Rule (Priority Rule #6):**
- **Prefer Server Actions** for all mutations and data fetching from client components.
- **Use Route Handlers only when:**
  1. Consumer is a webhook (UploadThing callback, Resend webhook)
  2. OAuth callback (Google OAuth redirect)
  3. File needs to be streamed (PDF export, image download)

**Decision Log:**
- `GET|POST /api/auth/[...nextauth]` — Required for Auth.js OAuth callbacks and session management ✓

### On New User Registration (both credentials + OAuth)

1. Create `User` record
2. Create `Family` ("Keluarga [FirstName]")
3. Create `FamilyMember` (role: ADMIN)
4. Create 12 default `Category` records (4 income + 8 expense)

### Security Decisions

- **bcrypt rounds: 10** — Balanced security vs. performance
- **Email enumeration prevention** — `forgotPasswordAction` always returns success
- **Token expiry: 1 hour** — VerificationToken expires in 3600000ms
- **Session maxAge: 30 days** — Database sessions (not JWT)
- **`allowDangerousEmailAccountLinking: true`** — Links Google OAuth to existing credentials account with same email

## 6. DATABASE SCHEMA (Current State)

**Status:** ✓ Complete — Full schema implemented in Milestone 2.

### Schema Design Principles

1. **UUID Primary Keys** — All tables use `@default(uuid())` for privacy and scalability
2. **Soft Deletes** — Critical tables (User, Family, Transaction) include `deletedAt` for audit trails
3. **Money as Decimal** — `Decimal @db.Decimal(19, 4)` avoids floating-point errors
4. **Strategic Indexes** — Composite indexes on foreign keys + frequently queried columns
5. **Enums for Fixed Sets** — Role, TransactionType, BudgetPeriod, BillStatus, NotificationType, RecurrenceFrequency

### Tables Overview

**Authentication & Users:**
- `User` — User accounts with credentials/OAuth
- `Account` — NextAuth.js OAuth provider accounts
- `Session` — Active user sessions
- `VerificationToken` — Email verification tokens
- `UserSettings` — User preferences (theme, language, currency)

**Family & Access Control:**
- `Family` — Family group container
- `FamilyMember` — Junction table (User ↔ Family) with role-based access (Admin/Parent/Child)

**Financial Entities:**
- `Category` — Transaction categories (income/expense)
- `Transaction` — Income/expense records (soft delete enabled)
- `Budget` — Monthly spending limits with denormalized `spent` field
- `SavingsGoal` — Savings targets with progress tracking
- `Bill` — Recurring bills with due dates and status
- `RecurringTransaction` — Templates for auto-generating transactions

**System:**
- `Notification` — In-app notifications (budget alerts, bill reminders, etc.)

### Key Relationships

```
User ─┬─< FamilyMember >─┬─ Family ─┬─< Transaction
      │                   │           ├─< Category
      ├─< Transaction     │           ├─< Budget
      ├─< Notification    │           ├─< SavingsGoal
      └─< UserSettings    │           ├─< Bill
                          │           └─< RecurringTransaction
                          └─ (Junction: Many-to-Many)
```

### Indexes Created

```prisma
// Composite indexes for common queries
@@index([familyId, date(sort: Desc)])   // Transactions by family + date
@@index([userId, isRead])                // Unread notifications
@@index([familyId, status])              // Bills by status
@@unique([familyId, userId])             // Prevent duplicate family membership
```
